How to view the history or log of your repository. Why is this useful?

git log

It is useful since it displays information about commits. You may feel like you need to go back to a previous commit, and this will help in such situation. It also allows to display the author of commits, so it helps tracking contributions. It also can show the branch tree, to get a general picture of branching and merging of versions.

Lab 10 History. (n.d.). Retrieved September 21, 2015, from http://gitimmersion.com/lab_10.html

Ensure you understand the states of objects in a repository (untracked, unmodified, modified, staged)

untracked: The file is not under git management.

unmodified:The file has no changes in working directory.

modified: The file in the working directory differs from the repository one.

staged: Changes have moved to the staging area, and are ready to be commited.

Get git to ignore certain files. Why do we want to do this?

The gitignore is ignoring the ignoreFolder. Atom displays ignored files in darker gray. Gitignore is simply ignoring the whole folder with "ignoreFolder/"

This is usefull when your project has files that do not need to be under git. Mostly files that are not source or are generated. Binary files usually do not use git tracking.

Check out how Git is supported in your favourite editor. Does it just show information or does it have support for Git commands?

Atom, shows information but does not allow to run commands directly. It shows information on the footer, about changes to the current file, and the current branch. It also shows the files on the file tree in different colors, if they have been modified since last commit. On the line number column, it displays colors when changes are made, staged, and for deletions.
